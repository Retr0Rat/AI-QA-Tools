## Overview
Training a model is only the beginning. This course teaches students to operationalize ML systems: automated training pipelines, model registries, A/B testing, monitoring for drift, and incident response. Students work with industry-standard MLOps platforms throughout.

## Tools & Technologies
- Python 3.x
- MLflow (experiment tracking and model registry)
- Kubeflow Pipelines
- Google Cloud Vertex AI
- Docker
- Kubernetes (basics)
- GitHub Actions (CI/CD)
- Evidently AI (data and model monitoring)
- FastAPI
- Prometheus and Grafana (metrics and dashboards)
- Terraform (infrastructure as code, introductory)
- Weights & Biases

## Topics
- The ML lifecycle: development to production
- Experiment tracking and model versioning with MLflow
- Automated training pipelines and DAGs
- Model serving: REST, batch, and streaming inference
- Containerizing ML models with Docker
- Kubernetes for ML workloads
- CI/CD pipelines for ML (GitHub Actions)
- Feature stores and training-serving skew
- Data drift and model performance monitoring
- A/B testing and canary deployments
- Model retraining triggers and automation
- Cloud ML platforms: Vertex AI on GCP

## Projects & Capstone

### Project 1: End-to-End MLOps Pipeline
Students take a trained model from a previous course and build a full MLOps pipeline: automated retraining triggered by data drift (detected with Evidently AI), model registration in MLflow, Docker-containerized API, and a GitHub Actions deployment workflow.

### Project 2: Monitoring Dashboard
Deploy a model to GCP Vertex AI with a Prometheus + Grafana monitoring stack. Students configure alerts for latency, error rate, and prediction distribution shift, and write a runbook for on-call response.
